# 缠论工程化规则

## 目标

把缠论从主观看图转换为可测试、可回测、可解释的规则引擎。系统宁愿保守、简化，也不输出无法复现的“玄学式”结论。

## 当前算法口径

| 阶段 | 规则 | 对应代码 |
| --- | --- | --- |
| 数据准备 | Date索引化，Open/High/Low/Close/Volume转数值 | `tradingagents/chan/analyzer.py` |
| 分型 | 三K顶底分型 | `detect_fractals` |
| 成笔 | 交替分型、最小间隔、价格方向验证 | `build_strokes` |
| 中枢 | 连续三笔价格区间重叠 | `build_centers` |
| 背驰 | 同向笔创新高/低且MACD面积或价格力度衰减 | `_latest_divergence` |
| 信号 | B1/S1、B2/S2、B3/S3 | `_signals` |
| 摘要 | 动作、评分、理由、止损止盈 | `_agent_summary` |
| 回测 | 下一根开盘执行，全仓示例，交易成本，止损 | `tradingagents/chan/backtest.py` |

## AI Agent输出规则

AI只能解释规则引擎给出的结构，不能替代规则引擎。

允许输出：

- 当前结构：分型、笔、中枢、买卖点。
- 当前动作：观察、试买、持有、减仓、平仓。
- 触发条件：突破、回抽、跌回中枢、止损价。
- 风险提示：级别冲突、背驰失败、回测交易次数不足。

禁止输出：

- 保证盈利、确定上涨、确定见底。
- 未回测策略的高确定性建议。
- 忽略A股T+1和涨跌停的即时交易建议。
- 未说明级别的结论。

## 可配置参数

| 参数 | 默认 | 影响 |
| --- | --- | --- |
| `min_stroke_bars` | 5 | 成笔越大，信号越少但更稳 |
| K线级别 | 日线 | 周线更稳，月线更慢 |
| 回测策略 | `combined` | 控制B/S信号集合 |
| 交易成本 | 0.12% | 覆盖手续费、印花税和滑点近似 |
| 止损 | 8% | 无中枢时的硬止损 |

## 后续增强路线

1. 加入K线包含关系处理，并与当前简化版并行回测。
2. 加入线段级别中枢，减少“只有笔级中枢”的噪声。
3. 接入多级别共振：周线定方向、日线找买卖点。
4. 加入板块强度过滤：只在强势板块中寻找三买。
5. 加入持仓管理：盈利后用中枢下沿或移动止损代替固定止损。

## 测试要求

- 修改缠论计算时必须运行 `pytest tests/test_chan_engine.py -q`。
- 修改页面时必须运行 `python -m py_compile web/pages/8_Chan_Agent.py`。
- 改变信号定义时必须记录回测对比结果。
