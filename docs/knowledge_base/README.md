# A股AI量化知识库

本知识库用于给系统、用户和后续开发者提供统一口径。它不是投资建议库，也不承诺任何策略收益；所有交易规则、因子、缠论信号都必须经过可复现回测、样本外验证和风控约束后才能进入模拟盘或实盘决策。

## 知识分层

| 层级 | 内容 | 用途 |
| --- | --- | --- |
| 理论层 | 缠论、技术指标、量化因子、市场微观结构 | 解释信号来源 |
| 规则层 | A股交易时间、涨跌停、T+1、申报单位、撮合机制 | 约束模拟交易和回测 |
| 策略层 | 买卖点、择时、选股、多因子组合 | 生成可回测策略 |
| 验证层 | 回测、滚动验证、样本外、交易成本、风险控制 | 判断策略是否可用 |
| Agent层 | 检索、提示词、结论边界、风险提示 | 让AI输出更稳定 |

## 文档规范

每篇知识文档应包含：

- `用途`: 这篇文档解决什么问题。
- `核心口径`: 系统采用的明确定义。
- `工程化规则`: 如何落到代码、数据和回测。
- `常见误区`: 避免AI或用户误用。
- `代码落点`: 对应模块、页面或测试。
- `来源`: 资料链接，优先官方、论文、成熟开源项目。

## 更新原则

1. 交易制度变化时，优先查交易所官方来源。
2. 策略定义变化时，必须同步测试或回测脚本。
3. AI解释可以扩写，但不能覆盖规则引擎的确定性输出。
4. 回测收益只能描述历史样本，不能写成未来收益承诺。
5. 对存在分歧的概念，保留“系统采用口径”和“其他解释”两栏。
