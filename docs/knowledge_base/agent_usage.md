# AI Agent使用规范

## 用途

规范系统中的AI荐股、缠论Agent、股票监控解释和模拟盘持仓建议。AI的职责是解释和组合信息，不是绕过规则引擎直接下判断。

## 检索顺序

当Agent生成交易建议时，优先读取：

1. 当前页面的实时数据、K线、因子、回测结果。
2. 本知识库中对应主题文档。
3. 代码产生的结构化信号。
4. 外部新闻、公告、政策资料。

## 输出模板

```text
结论：观察 / 试买 / 持有 / 加仓 / 减仓 / 平仓
级别：日线 / 周线 / 月线 / 多级别
依据：最多3条，引用结构化信号或因子
触发条件：价格、时间、结构或风控条件
风险：至少1条
边界：仅用于研究和模拟盘，不构成投资建议
```

## 必须遵守

- 明确分析级别和数据日期。
- 明确是否有回测支持。
- 明确止损或结构失效条件。
- 高不确定性时输出“观察”，不要强行给买入。
- 策略冲突时解释冲突，而不是只挑支持买入的一边。

## 禁止输出

- “稳赚”“必涨”“确定反转”“满仓买入”。
- 不解释风险的荐股。
- 忽略回测低交易次数或高回撤的建议。
- 忽略A股T+1、涨跌停和成交约束。
- 把AI主观判断当作已经验证的策略。

## 和规则引擎的关系

| 来源 | 角色 | 优先级 |
| --- | --- | --- |
| 交易规则 | 硬约束 | 最高 |
| 回测结果 | 验证依据 | 高 |
| 因子/缠论信号 | 策略输入 | 高 |
| AI解释 | 归纳和表达 | 中 |
| 新闻/情绪 | 辅助信息 | 中低 |

## 示例

合格输出：

> 结论：观察偏试买。日线级别出现三买候选，但回测最大回撤较高，建议只在回抽不跌回中枢且成交量不异常放大时小仓试错；若跌回中枢或触发止损，应退出观察。

不合格输出：

> 已经出现三买，明天必涨，可以重仓。
